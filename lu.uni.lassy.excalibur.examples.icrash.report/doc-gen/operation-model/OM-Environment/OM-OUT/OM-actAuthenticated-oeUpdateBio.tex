\subsection{Operation Model for oeUpdateBio}

\label{OM-oeUpdateBio}


The \msrcode{oeUpdateBio} operation has the following properties:

	\begin{operationmodel}
	\addheading{Operation}
	\adddoublerow{oeUpdateBio}{updates biometric information for current actor }

	\addrowheading{Parameters}
	\addnumbereddoublerow{}{AdtLogin: dtLogin}{login of actor which request an update of biometric information} 
	\addnumbereddoublerow{}{AdtBiometric: dtBiometric}{new biometric information of actor} 
	\addnumbereddoublerow{}{AdtLogin: dtLogin}{ } 
	\addnumbereddoublerow{}{AdtBiometric: dtBiometric}{ } 

	\addrowheading{Return type}
	\addsinglerow{ptBoolean}

	\addrowheading{Pre-Condition (protocol)}
	\addnumberedsinglerow{PreP}{the system is started}
	\addnumberedsinglerow{PreP}{the actor is currently logged in ! (i.e. the associated ctAuthenticated instance is considered logged)}
		
	\addrowheading{Pre-Condition (functional)}
	\addnumberedsinglerow{PreF}{ }

	\addrowheading{Post-Condition (functional)}
	\addnumberedsinglerow{PostF}{a bio update confirmation message is sent to the actor}

	\addrowheading{Post-Condition (protocol)}
	\addnumberedsinglerow{PostP}{the biometric information of current actor is updated ! (i.e. new value of field 'bio' in the associated ctAuthenticated instance) }
	\end{operationmodel}
